# Good Practices for Software Development - Review Answers

## Question 1: Which point(s) were new to you?

The points that were new to me were:

* The importance of over-communicating with the team, especially when facing delays, technical issues, or changes in requirements.
* Using tools such as Github Gists, CodeSandbox, and screencasts to ask better technical questions.
* Joining meetings a few minutes early to build better relationships with teammates.
* The idea of preserving attention through deep work by blocking distractions and tracking productivity.

## Question 2: Which area do you think you need to improve on? What are your ideas to make progress in that area?

I need to improve on asking questions more effectively when I get stuck. Sometimes I ask for help without providing enough context about the problem or the solutions I have already tried.

To improve, I will:

* Clearly describe the issue and include error messages.
* Mention the approaches I have already tried.
* Share screenshots or code snippets whenever necessary.
* Spend some time researching the problem before asking for help.
* Organize my questions so that teammates can understand and answer them quickly.

I also want to improve my focus during work by reducing distractions, keeping my phone on silent, and dedicating specific time blocks for concentrated work.
